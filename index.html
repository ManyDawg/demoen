10. Name resolution infrastructure

Задание: Настройте инфраструктуру разрешения доменных имён для офисов HQ и BR: основной DNS-сервер реализован на HQ-SRV, сервер должен обеспечивать разрешение имён в сетевые адреса устройств и обратно в соответствии с таблицей 3, в качестве DNS сервера пересылки используйте любой общедоступный DNS сервер (77.88.8.7, 77.88.8.3 или другие).

Configure DNS on HQ-SRV:

dnf install -y bind bind-utils

nano /etc/named.conf

Add to "options":

listen-on port 53 { 127.0.0.1; 192.168.100.2; };

allow-query { any; };

forwarders {77.88.8.8; 8.8.8.8; 1.1.1.1; };

recursion yes;

Remove duplicate parameters in the file, leaving only one of each.

At the end of /etc/named.conf, define our zones:

zone "au-team.irpo" IN { type master; file "/var/named/au-team.irpo.zone"; }; zone "100.168.192.in-addr.arpa" IN { type master; file "/var/named/rev.100.zone"; }; zone "200.168.192.in-addr.arpa" IN { type master; file "/var/named/rev.200.zone"; };

[Screenshot needed: /etc/named.conf file with configured zones]

Now the zone files themselves:

nano /var/named/au-team.irpo.zone

6. Logging with rsyslog

Задание: Реализуйте логирование при помощи rsyslog на устройствах HQ-RTR, BR-RTR, BR-SRV: сервер сбора логов расположен на HQ-SRV, убедитесь что сервер не является клиентом самому себе, приоритет сообщений должен быть не ниже warning, все журналы должны находиться в директории /opt, для каждого устройства должна выделяться своя поддиректория которая совпадает с именем машины. Реализуйте ротацию собранных логов на сервере HQ-SRV: ротируются все логи находящиеся в директории и поддиректориях /opt, ротация производится один раз в неделю, логи необходимо сжимать, минимальный размер логов для ротации — 10МБ.

HQ-SRV — log server:

dnf install -y rsyslog

Uncomment these lines in /etc/rsyslog.conf:

#module(load="imudp")

#input(type="imudp" port="514")

#module(load="imtcp")

#input(type="imtcp" port="514")

[Screenshot needed: uncommented imudp and imtcp modules in /etc/rsyslog.conf]

At the end of the file, specify the log save path:

$template RemoteLogs,"/opt/%HOSTNAME%/%PROGRAMNAME%.log" *.warning ?RemoteLogs

[Screenshot needed: added RemoteLogs template in /etc/rsyslog.conf]

mkdir -p /opt

systemctl restart rsyslog

[Screenshot needed: restarted rsyslog (systemctl status rsyslog)]

rsyslog clients (HQ-RTR, BR-RTR, BR-SRV):

dnf install -y rsyslog

nano /etc/rsyslog.conf

*.warning @@192.168.100.2:514

[Screenshot needed: client rsyslog config (*.warning @@192.168.100.2:514)]

systemctl restart rsyslog

HQ-SRV must not have a line with @@192.168.100.2:514, as the server must not send logs to itself.

Log rotation on HQ-SRV:

nano /etc/logrotate.d/opt-logs

/opt/*/*.log { weekly rotate 4 compress missingok notifempty minsize 10M sharedscripts postrotate /bin/kill -HUP `cat /var/run/syslogd.pid 2>/dev/null` 2>/dev/null || true endscript }
